Noted - Android

My take on the classic note taking app.

Simple in functionality, this project has been designed to demonstrate the core principles and recommended best practices of application architecture, set out by the developers.android.com documentation.

This has involved the use of a view model to bridge the UI & data layers, as well as hold the state of the UI during recomposition. A data repository has also been implemented to further separate our Room database from the UI layer and ensure data integrity. This approach to architecture helps to promote stability & scalability.

The use of Kotlin coroutines allows for potential time-intensive tasks, such as data retrieval to be performed on background threads. This helps to improve the user experience by greatly reducing the chance of freezing UI, as this is updated on the main thread.

One feature I am pleased to have implemented is the undoing of deleted note entries through the use of an action button on the ‘snackbar’ displayed after choosing to delete a note. This is achieved by caching a copy of the note object upon deletion which can then be quickly ‘upserted’ again by the event handler attached to the action button.
